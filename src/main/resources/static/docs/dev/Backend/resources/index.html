Backend - Recursos

Esta sección explica lo imprescindible para que, al clonar el proyecto, sepas qué archivos/configurar y qué NO viene en el repo.
La guía detallada para levantar el servicio está en “Despliegue del Backend”.

¿Qué viene y qué no viene?​

Incluido en src/main/resources/:

  • application.properties → configuración por defecto para desarrollo local (incluye conexión a PostgreSQL con credenciales de ejemplo).
  • logback.xml → configuración de logs (consola; en despliegue puedes activar archivo rotativo).
  • templates/ (Thymeleaf) → páginas informativas (index.html, proyecto.html, quienes-somos.html, contactos.html y fragmentos en _fragments/).
  • static/ → estilos e imágenes que sirven esas páginas (css/, images/, web/).
  • registro.postman_collection → colección Postman para probar endpoints.

NO incluido (intencional):

  • Credencial de Firebase (FCM): el archivo JSON de servicio no se versiona.

    Si no lo agregas, los endpoints/tareas de Notificaciones PUSH fallarán con error de credenciales.

application.properties — ¿qué tocar al clonar?​

El archivo trae valores listos para local. En la práctica, al clonar normalmente solo ajustas estas 3 líneas:

spring.datasource.url=jdbc:postgresql://localhost:5432/db-clias
spring.datasource.username=postgres
spring.datasource.password=admin

  • Si usas otro puerto/usuario/BD, cámbialos aquí para tu entorno local.

  • Para producción, no edites este archivo: las credenciales las provee el sistema (variables de entorno).

  • Ver la página “Despliegue del Backend” donde se muestran los environment: para Docker Compose.

nota

Nota rápida sobre otros parámetros:

  • spring.jpa.hibernate.ddl-auto=update está pensado para desarrollo. En producción se recomienda none + migraciones.

  • Los timeouts de Hikari en este properties son bajos para prod; eso se ajusta en despliegue (no aquí).

Introducción a templates/ y static/ (Web mínimas)​

El backend sirve unas páginas informativas con Thymeleaf (sin relación directa 1:1 con tablas).
Rutas públicas principales (ver PagesController):

RutaPlantillaPropósito
/templates/index.htmlPágina de inicio
/proyectotemplates/proyecto.htmlInformación del proyecto
/quienes-somostemplates/quienes-somos.htmlEquipo/Institución
/contactostemplates/contactos.htmlContacto y soporte
/docsdocs/index.htmlDocumentación Oficial del proyecto
/webweb/index.htmlPágina para administradores

Los estilos e imágenes se ubican en static/ y se sirven automáticamente (propiedad activa): spring.web.resources.static-locations=classpath:/static/

⚠️ Introducción: esta sección solo contextualiza la carpeta web mínima.
En la siguiente hoja “Web (Thymeleaf) & Static Assets” se detalla estructura de plantillas, fragmentos,etc.

Logging​

  • logback.xml ya viene configurado para mostrar logs en consola.

  • En producción puedes activar archivo rotativo diario y enviar a un directorio (LOG_DIR).
