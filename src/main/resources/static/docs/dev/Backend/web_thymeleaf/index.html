Backend - Web (Thymeleaf) & Static Assets

Esta sección describe cómo está organizada y cómo mantener la parte web “ligera” servida por el backend:

  • Plantillas Thymeleaf para páginas informativas.
  • Assets estáticos (CSS, imágenes y páginas HTML puras).
  • Buenas prácticas para agregar/editar páginas sin romper estilos ni rutas.

Esta web es un site informativo y de soporte para el proyecto.

Estructura​

Estructura de src/main/resources/ (actual)​

resources/
├─ static/
│  ├─ css/
│  │  ├─ pages/
│  │  └─ site.css
│  ├─ images/
│  ├─ web/
│  └─ favicon.ico
├─ templates/

│  ├─ contactos.html
│  ├─ index.html

├─ logback.xml
└─ registro.postman_collection

Spring ya expone static/ gracias a
spring.web.resources.static-locations=classpath:/static/.

Páginas con Thymeleaf​

Controladas por PagesController

  • Rutas: /, /proyecto, /quienes-somos, /contactos.
  • Ventajas: fragmentos reutilizables, variables de modelo, i18n si aplica.

Páginas estáticas puras​

  • Rutas: /web/, /docs/.

  • Ventajas: cero lógica, ideales para micrositios de administración o documentación generada.

Fragmentos y layout (versión final)​

¿Por qué este enfoque?​

Usamos Thymeleaf + fragmentos (head, navbar, footer) para mantener el micrositio del backend simple y profesional:

  • DRY: una sola fuente de verdad para <head>, navegación y footer.
  • Consistencia: misma marca y estructura en todas las páginas.
  • Bootstrap listo: estilos y JS se cargan en los fragmentos (CDN), sin build extra.
  • Extensible por página: el parámetro extra de head que permite inyectar CSS/metas/script solo en la vista que lo necesita.
  • Responsivo: el navbar usa el patrón “hamburguesa + dropdown-split” de Bootstrap y el footer carga bootstrap.bundle.min.js.

Fragmentos​

  • Head — título, favicon, Bootstrap, fuente y slot extra.
  • Navbar — logos, enlaces, dropdown con “split” y estado activo.
  • Footer — créditos y JS de Bootstrap (bundle incluye Popper):

¿Cómo se usa en una página?​

a) Con extra vacío (solo layout + global):

<!-- templates/index.html -->
<html lang="es">
  <head th:replace="_fragments/head :: head('CLIAS | Inicio', null)"></head>
  <body>
    <header th:replace="_fragments/navbar :: navbar(${active})"></header>
    <main class="container">...</main>
    <footer th:replace="_fragments/footer :: footer"></footer>
  </body>
</html>

b) Inyectando CSS (u otros nodos) solo para esta página con extra:

<!-- templates/proyecto.html -->
<!-- 1) Declaras un fragmento local con lo que quieras inyectar -->
<th:block th:fragment="pageExtra">
  <link rel="stylesheet" th:href="@{/css/pages/proyecto.css}"/>
  <!-- También podrías añadir metas específicas:
       <meta name="description" content="Proyecto CLIAS"> -->
</th:block>

<!-- 2) Lo pasas como segundo parámetro -->
<html lang="es">
  <head th:replace="_fragments/head :: head('CLIAS | Proyecto', ~{::pageExtra})"></head>
  <body>
    <header th:replace="_fragments/navbar :: navbar(${active})"></header>
    <main class="container">...</main>
    <footer th:replace="_fragments/footer :: footer"></footer>
  </body>
</html>

Extender/editar sin romper nada​

  • Nueva página Thymeleaf
  1. Crea templates/faq.html.
  2. (Opcional) Declara pageExtra con faq.css y pásalo al head.
  3. En el controlador: 
    @GetMapping("/recursos")
       fun faq(model: Model): String {
           model.addAttribute("active", "faq")
           return "faq"
       }

  4. Agrega el enlace en navbar.html si debe aparecer en el menú.
  • Nuevo item en el dropdown “Proyecto”
    Añade una línea dentro de <ul class="dropdown-menu"> (ejemplo Preguntas Frecuentes): 
    <li><a class="dropdown-item" th:href="@{/preguntas-frecuentes}">Preguntas Frecuentes</a></li>
