Frontend Web - Despliegue de la App Web (Flutter Web)

Esta guía explica cómo construir y desplegar la app web (frontend hecho en Flutter Web) junto al backend de IA - VPH.

Requisitos​

  • Flutter SDK instalado y configurado para web (Chrome o Edge).
  • Git 2.40+
  • IDE recomendado: VS Code (aunque puedes usar cualquier IDE).

Verificar entorno Flutter

flutter doctor
flutter --version
flutter devices   # Debe listar "Chrome" / "Edge" como dispositivo web

Ejecución del proyecto en Local​

Repositorio del frontend web:

git clone https://github.com/midaro99/telemedicina_web.git
cd telemedicina_web

Instalar dependencias y lo ejecutamos

flutter pub get
flutter run

Compilación para Producción​

Build con base path

flutter build web --base-href /web/

¿Por qué --base-href /web/? Porque la app no se servirá desde la raíz (/), sino desde la ruta /web/ dentro del backend (Spring Boot la servirá desde src/main/resources/static/web).

  • Flutter genera rutas y assets relativos al <base href="/web/">.

  • Si no usas --base-href /web/, el navegador intentará cargar /main.dart.js, /assets/... desde la raíz del dominio y dará 404.

  • Con --base-href /web/, todos los assets quedan referenciados correctamente como /web/main.dart.js, /web/assets/....

Al finalizar, Flutter genera la carpeta:

telemedicina_web/build/web

Ajustes Necesarios de assets del build​

Dependiendo de la versión de Flutter y cómo estén declarados los assets en pubspec.yaml, es posible que el build produzca una estructura anidada assets/assets/.... Para el despliegue en nuestro backend, normalizamos estas rutas:

  1. Mover fuentes
  • Origen: build/web/assets/assets/fonts/
  • Destino: build/web/assets/fonts/
  1. Mover imágenes
  • Origen: build/web/assets/assets/images/
  • Destino: build/web/assets/

Si alguna carpeta no existe (por ejemplo, no hay images), simplemente omite ese paso.

Integración con el backend​

Nuestra app web se servirá desde el backend (Spring Boot), en la ruta /web.

  1. Ir a la carpeta estática del backend <BACKEND>/src/main/resources/static/web
  2. Reemplzar Archivos Copia el contenido de: telemedicina_web/build/web dentro de <BACKEND>/src/main/resources/static/web
  • Reemplaza los archivos existentes (siempre son 12).
  1. Reiniciar/Reconstruir el backend
  • Si corres en local desde IntelliJ, basta con re-ejecutar.
  • Si usas Docker en producción, reconstruye la imagen del backend y levántala.

Verificación​

  • Local: http://localhost:8080/web
  • Producción: https://clias.ucuenca.edu.ec/web

Problemas comunes​

ErrorPosible causaSolución
404 en assetsNo usaste --base-href /web/Reconstruir con la opción correcta
assets/assets/... duplicadosVersión de FlutterMover manualmente a assets/
Error CORS en llamadas al backendBackend no acepta origen del frontendConfigurar CORS en Spring Boot
401 UnauthorizedToken JWT expirado o ausenteReautenticar al usuario

Recomendaciones de seguridad​

  • Servir siempre bajo HTTPS en entornos productivos.
  • Restringir CORS solo a los dominios del frontend autorizado.
  • Mantener separados los entornos (local vs producción).
